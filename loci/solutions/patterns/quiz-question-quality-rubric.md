# Quiz Question Quality Rubric

Extracted from dokime (GARP RAI quiz skill, retired Apr 2026). Generalises to any drill/quiz skill.

## Question Construction

- **One concept per question.** If the user gets it wrong, you need to know exactly which concept failed.
- **One unambiguous keyed answer.** If two options could arguably be correct, rewrite.
- **Scenario-based where possible.** "A bank uses...", "An analyst is...", "A company is..."
- **No attribution questions.** Never test "who coined X", "which year", "which institution."
- **Trap-based distractors.** At least one wrong answer targets a documented common confusion.
- **No "all of the above" / "none of the above"** — lazy and untestable.

## Option Format (Two Valid Patterns)

1. **Identify-the-term**: options are just term names. Use when testing if you can name the right concept.
2. **Describe-the-behaviour**: options are full sentences. Use when testing understanding.
3. **NEVER mix**: "Term — [explanation]" in a single option is always wrong.

## Mode Switching by Accuracy

- <60%: strict MCQ, explain each distractor after answer
- 60-69%: free-recall (no options presented)
- ≥70%: maintenance MCQ, lighter touch
- >80% across 3+ sessions: hard scenario mode, no options, free-form

## Grading

- Ground correctness against source material, not general reasoning.
- Infer confidence from response style (confident / unsure / guess).
- If question was invalid, void the record immediately.
