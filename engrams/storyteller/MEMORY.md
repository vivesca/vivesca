# Storyteller Agent Memory

## CRO Framing Patterns

**HK insurer, compliance AI pilot (Mar 2026)**
- Hallucination rate (12%) was the number that would dominate the room — led with it, reframed it as "1 in 8 answers may contain unsupported claims"
- Retrieval accuracy and latency: dropped entirely. CRO does not care.
- Answer faithfulness (91%): kept only as implicit context for the error rate, not as a headline
- The tension that landed: failure mode concentrated on complex queries = exactly where compliance value is highest. Naming that tension proactively is more credible than burying it.
- Control framing that worked for CRO approval ask: (1) human sign-off required on every output, (2) complete audit trail, (3) explicit out-of-scope list, (4) numeric exit condition (>5% live error rate = pause)
- Competitive framing: "peer insurers in HK are already running comparable tools" — used to answer "why move at all?" without overselling the upside

## Hard Rules Reinforced
- Never lead with accuracy metrics for a CRO. Lead with the failure rate and the control for it.
- "1 in N" beats percentages for executive audiences — more visceral.
- The ask must name who is accountable and by when. A 1-pager without an owner is not approvable.
- If the technical content has a tension built in (value case = complex queries; failure mode = complex queries), name it. Burying it costs credibility; surfacing it earns it.
