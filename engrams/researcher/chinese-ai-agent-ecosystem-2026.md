## 3. Qwen-Agent Framework

### Architecture
- Open source: github.com/QwenLM/Qwen-Agent (built upon Qwen≥3.0)
- Components: BaseChatModel (LLMs with function calling), BaseTool, Agent base classes
- Features: function calling, MCP, code interpreter, RAG, Chrome extension
- Recommendation: use Qwen-Agent for agentic Qwen3 — encapsulates tool-calling templates + parsers, reduces complexity
- Native vLLM tool call parsing support added Jul 2025
- DeepPlanning benchmark open-sourced Jan 2026

### Key Pattern
- Tool-calling template encapsulation: developers don't implement parsing; framework handles format transparently
- Qwen3 specifically trained for tool calling — framework and model co-designed (tighter than Western LLM/framework separation)

## 4. Chinese Multi-Agent Frameworks

### MetaGPT / MGX
- Core philosophy: "Code = SOP(Team)" — encode human SOPs into prompt sequences for agent teams
- MGX launched Feb 19, 2025: world's first AI agent dev team platform
- Five specialized agents: Mike (PM/coordinator), Emma (product manager), Bob (architect), Alex (engineer), David (data analyst)
- SOPs produce structured artifacts: PRDs, system design diagrams, API specs — not just code
- AFlow paper: ICLR 2025 oral (top 1.8%), #2 in LLM-based Agent category — automated workflow design via Monte Carlo search
- OpenManus: MetaGPT team built in 3 hours as OSS Manus alternative; 16K GitHub stars in 2 days
- FoundationAgents org on GitHub: github.com/FoundationAgents/MetaGPT

### AgentScope 1.0 (Alibaba)
- Paper: arxiv:2508.16279, Aug 2025
- Key differentiators vs LangGraph/CrewAI:
  - **Group-wise tool management**: bundles related tools into task-oriented cohorts (addresses "paradox of choice" — tool abundance degrades agent performance)
  - **Dual stateful/stateless MCP clients**: stateful for session-dependent services (browser), stateless for transactional lightweight services
  - **StateModule**: automated compositional state management — save/restore entire nested agent hierarchies
  - **Non-invasive hooks**: pre/post hooks at lifecycle points (reply, observe, reasoning, acting) without altering core
  - **Trajectory comparison in evaluation**: side-by-side visual comparison of divergent execution paths
  - **Statistical distribution visualization**: performance as probability distributions with bootstrapping, not single-point metrics
  - **Explicit interruption handling**: context-preserving (not mere cancellation)
  - **Meta Planner**: hierarchical task decomposition built in
- Auto-deploys agents as FastAPI services with health monitoring
- Java version released; Spring AI Alibaba core upgraded to AgentScope
- GitHub: github.com/agentscope-ai/agentscope

### CAMEL
- Original: role-playing for multi-agent "society" simulation
- "Inception prompting": carefully designed constraints to prevent role-flipping ("Never flip roles!", "Never instruct me!")
- OASIS (2025): social media simulator, up to 1M agents, studies information spread, group polarization
- Finding scaling laws of agents (self-described mission): github.com/camel-ai/camel
- Data generation: 25K conversation AI Society dataset

### Key Chinese vs Western Framework Differences
- Chinese frameworks emphasize **production deployment as first-class** (AgentScope auto-FastAPI, Java version)
- **Co-designed model + framework** pattern (Qwen-Agent co-designed with Qwen3; vs LangChain as model-agnostic)
- **SOP encoding** as explicit architecture (MetaGPT) — Western frameworks typically don't encode procedural knowledge
- **Group-wise tool management** (AgentScope) — Western frameworks present flat tool lists
- Message hub / group conversation topology emphasized equally to hierarchical (different use-case distribution)

## 7. Manus — Technical Analysis

### Architecture (from Ji Yichao's July 2025 blog)
- Multi-model backend: Claude, Qwen, proprietary systems
- CodeAct: actions executed via Python code generated by LLMs (not JSON tool calls)
- State machine for tool availability (not retrieval-based tool loading)
- ~50 tool calls average per task — explicit attention management required

### 7 Core Context Engineering Techniques
1. **KV-cache hit rate optimization**: stable prefixes, append-only context, session ID routing across distributed inference workers (10x cost difference cached vs uncached)
2. **Logit masking (Mask, Don't Remove)**: mask token logits rather than removing tools from context — removing tools breaks KV-cache
3. **File system as externalized memory**: unlimited size, persistent, agent-operable; store paths not content
4. **Attention manipulation via recitation**: agents rewrite todo.md to push recent objectives into attention window ("lost-in-the-middle" mitigation at 50+ tool calls)
5. **Restorable compression**: store URLs/paths instead of content; recover on-demand
6. **Error preservation**: retain failed actions and stack traces — models learn failure patterns implicitly
7. **Diversity injection**: controlled variation in serialization templates to prevent behavioral mimicry

### Named Techniques (novel)
- "Stochastic Graduate Descent": their term for manual iterative architecture searching + empirical testing
- 100:1 input-to-output token ratio: agents have vastly larger prefills than typical chatbots

### Chinese Developer Reaction
- Manus defined the "product paradigm of Agents" for Chinese practitioners
- Subsequent Chinese agents (Lovart, YouWare, Fellou 2.0) all use Manus UI pattern: thought chain + visual execution panel

### Industry Patterns (China FS)
- 2025 banking IT investment: RMB 198.4B; agent-related: 23% share
- 14 major banks: "AI+Finance" → "Human+AI" reframing (mid-2025)
- DeepSeek deployment: state-owned + joint-stock banks across customer service, operations, credit approval
- PBoC "AI + Finance" framework (2025): national blueprint for AI in financial reform
- EY white paper: "AI Bank White Paper — Agent-Driven Financial Paradigm Reconstruction"
- Projected: 70% of financial institutions deploy multi-agent coordination systems by 2027

### China FS vs Western FS — Key Differences
- **No SR 11-7 equivalent**: China uses "end-to-end, in-depth supervision" framework; algorithm filing requirements (mandatory registration); fairness audits
- **DeepSeek as FS backbone**: state banks integrating open-source DeepSeek at scale; no Western equivalent (banks don't run Meta Llama at core banking scale)
- **Regulatory driver is PBoC/CBIRC**, not Fed/OCC — different accountability structure
- **Explainability requirement** exists but "black box" prohibition is softer in practice for internal risk tools vs customer-facing
- **Human accountability clause**: institutions "must manually review monitoring results and bear ultimate responsibility" — similar spirit to four-eyes but written into RegTech rules
- Three national standards on GenAI security (Apr 25, 2025, effective Nov 1, 2025): data training fairness, algorithmic explainability, opt-out mechanisms
